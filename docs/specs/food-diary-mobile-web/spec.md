# 美食日记移动端网站

## 用户故事
作为一个美食爱好者
我想要一个能记录和展示自己制作美食的网站
以便以炫酷的方式回顾和分享我的烹饪成果

## 背景
用户希望建立一个公开展示的美食记录网站，用于记录日常制作的美食。核心诉求是：
- 纯静态网站，无需后端服务器
- 内容管理通过 Git 提交，图片存储在仓库内，不依赖外部图床
- 移动端优先，支持炫酷的 3D 展示效果
- 部署到 GitHub Pages，实现零成本托管

## 功能需求

### 核心功能
- **美食记录展示**：以卡片形式展示美食，包含名称、描述、标签、日期、照片
- **多种展示模式**：
  - 时间线卡片流（默认）
  - 瀑布流布局
  - 3D 卡片浮动效果
  - 3D 圆柱/球体展示
- **标签筛选**：点击标签可筛选出对应分类的美食
- **深浅色模式切换**：支持跟随系统或手动切换
- **响应式设计**：移动端优先，同时适配桌面端

### 内容管理
- 美食数据以 Markdown 文件存储在仓库中
- 图片直接存储在仓库的 `images/` 目录
- 通过 Git 提交添加新内容，GitHub Actions 自动构建部署

### 数据结构
每条美食记录包含：
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| name | string | 是 | 美食名称 |
| description | string | 是 | 美食描述 |
| tags | string[] | 是 | 分类标签，如"家常菜"、"甜品"、"日式" |
| date | string | 是 | 制作日期，格式 YYYY-MM-DD |
| image | string | 是 | 图片路径，相对于 images/ 目录 |

### 边界条件
- 无记录时显示空状态引导
- 图片加载失败时显示占位图
- 标签不存在时显示"暂无此分类"
- 移动端小屏幕下 3D 效果应可关闭以节省性能

### 异常处理
- 网络异常时的离线提示
- 图片懒加载失败的重试机制
- 3D 渲染不支持时降级为普通展示

## 技术考量

### 技术栈
- **框架**：React + Vite（静态站点生成）
- **样式**：Tailwind CSS 或 CSS Modules
- **3D 效果**：Three.js / React Three Fiber 或 CSS 3D transforms
- **部署**：GitHub Pages + GitHub Actions

### 性能优化
- 图片懒加载
- 图片压缩（构建时处理或提交前压缩）
- 3D 效果按需加载
- 移动端可选关闭重度 3D 效果
